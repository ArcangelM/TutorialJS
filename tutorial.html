Historia JavaScript

Conozcamos como nacio este lenguaje de programacion

1 Mayo 1995Primer versión del lenguaje
Netscape Navigator (dominante de la web en ese momento) decide agregar interactividad en las paginas HTML con un lenguaje de programación liviano. Contrata a Brendan Eich para implementarlo y termina su primer versión en 10 días con el nombre de Mocha.
1 Septiembre 1995 Nuevo nombre: LiveScript
1 Diciembre 1995 Netscape y Sun hacen un acuerdo de licencia para darle el nombre Javascript.
El nombre JavaScript lograba el roll original del lenguaje: Java de Sun proveia la construcción de bloques de gran escala para la Web, mientras que JavaScript era el "pegamento" conectando los bloques.
1 Noviembre 1996 Inicio del ECMA-262
Netscape decide estandarizar JavaScript, lo cual Microsoft pide a Ecma International (Organizacion de Estandares) para que lo hospede con el nombre ECMA 262.
1 Enero 1997 HTML dinámico
El HTML Dinámico o DHTML (del inglés Dynamic HTML) designa el conjunto de técnicas que permiten crear sitios web interactivos utilizando una combinación de algún lenguaje de marcado estático (como HTML), un lenguaje interpretado en el lado del cliente (como JavaScript), el lenguaje de hojas de estilo en cascada (CSS) y la jerarquía de objetos de un Document Object Model (DOM).
1 Junio 1997 Primer edicion del ECMA-262: ECMAScrip
La primer release del ECMA-262 sale a la luz, pero como Sun (hoy Oracle) poseia la marca Java el lenguaje no podía ser estandarizado bajo el nombre JavaScript. Por lo que se estandariza con el nombre ECMAScript. Sus implementaciones son oficialmente llamdas JavaScript, JScript, etc.
1 Enero 2001 JSON
Douglas Crockford define y nombra Javascript Object Notation (JSON), lo que iniciaba una nueva forma de intercambio y transferencia de datos basada en JavaScript.
1 Enero 2005 AJAX
Asi fué llamado por Jesse James Garrett luego de la salida de Google Maps (uno de los pioneros en este tipo de aplicaciones). AJAX lograba el estilo de una aplicación de escritorio en cuanto a transferencia de datos utilizando XMLHttpRequest para las peticiones asincronicas y luego actualizando el contenido con HTML Dinámico.
1 Enero 2006 Nace jQuery
Con el objetivo de ayudar a la manipulación del DOM, abstrayendose de las diferencias entre exploradores y logrando que "jugar" con el DOM sea algo divertido.
1 Enero 2009 NodeJS - JavaScript en el Servidor
Ryan Dahl crea un framework para aplicaciones de red por arriba de la maquina virtual de javascript Google V8. Logrando que se pueda utilizar JavaScript del lado del servidor.

Caracteristicas JavaScript

Conozcamos ahora las características de JavaScript que haces de este lenguaje, uno de los más populares en la actualidad.

Imperativo y estructurado

JavaScript es compatible con gran parte de la estructura de programación de C (por ejemplo, sentencias if, bucles for, sentencias switch, etc.).

Prototipado

JavaScript usa prototipos en vez de clases para el uso de herencia. Es posible llegar a emular muchas de las características que proporcionan las clases en lenguajes orientados a objetos tradicionales por medio de prototipos en JavaScript.

Orientado a Objetos

Maneja objetos dentro de nuestra página Web y sobre ese objeto podemos definir diferentes eventos. Dichos objetos facilitan la programacion de paginas interactivas, a la vez que se evita la posibilidad de ejecutar comandos que puedan ser peligrosos para la maquina del usuario, tales como formateo de unidades, modificar archivos etc.

Dinámico

Es dinámico, responde a eventos en tiempo real. Eventos como presionar un botón, pasar el puntero del mouse sobre un determinado texto o el simple hecho de cargar la página o caducar un tiempo. Con esto podemos cambiar totalmente el aspecto de nuestra página al gusto del usuario.

Seccion #2 (Variables)

Declaración de Variables

Codigo Ej.

              
          		var a=8;
          		b = 10;
          		let y = 5;
Resumen

Pudes declarar de 3 maneras

var a=8.Esta sintaxis puede ser usada para declarar tanto variables locales como globales.

b=10. Esto siempre declara una variable global y no puede ser cambiada a nivel local.

let y = 5. Esta variable puede ser usada para declarar una variable local en un bloque de ámbito.

Evaluacion de Variables

Codigo Ej.

              	var a;
                console.log('El valor de a es ' + a); // El valor de a es undefined

                console.log('El valor de b es ' + b); // El valor de b es undefined
                var b;
                console.log('El valor de c es ' + c); // Uncaught ReferenceError: c no está definida
								
                let x;
                console.log('El valor de x es ' + x); // El valor de x es undefined

                console.log('El valor de y es ' + y); // Uncaught ReferenceError: y no está definida
                let y;
Resumen

Consideraciones

Una variable declarada usando la sentencia var o let sin asignarle un valor inicial tiene el valor undefined.

Al intentar acceder a una variable no declarada dará como resultado la excepción ReferenceError.

Se puede usar undefined para determinar si una variable tiene un valor.

Ámbito de una Variable

Codigo Ej.

              	if (true) {
        			     var x = 5;
        			  }
			          console.log(x); // x vale 5
              	if (true) {
                  let y = 5;
                }
                console.log(y);  // ReferenceError: y no está definida
Resumen

Consideraciones

Cuando declaras una variable fuera de una función, esta es llamada variable global, porque esta disponible para cualquier otro código en el documento actual. Cuando declaras una variable dentro de una función, esta es llamada variable local, porque está disponible solo dentro de esa función.

en elcódigo el primer log registrará 5, porque el ámbito de x es la función dentro del cual se declara x, no el bloque, que en este caso es la sentencia if.

el anterior comportamiento cambia, cuando usamos la declaración let

Utilice (switch) para especificar muchos bloques alternativos de código para ser ejecutado

Seccion #4 (Ciclos)

Ciclo For

Codigo Ej.

              	for (i = 0; i < 5; i++) {
            		  text += "The number is " + i + "
";
            		}
Resumen

For

El bucle for es a menudo la herramienta que va a utilizar cuando se quiere crear un bucle.

El bucle tiene la siguiente sintaxis:

for (statement 1; statement 2; statement 3) {

code block to be executed

}

Bucle While y Do/While

Codigo Ej.

              	//Bucle while
            		while (i < 10) {
            		    text += "The number is " + i;
            		    i++;
            		}
            		
            		//Bucle do While
            		do {
            		    text += "The number is " + i;
            		    i++;
            		}
            		while (i < 10);
Result

While y Do/While

En el bucle WHILE se ejecuta un bloque de código siempre que una condición especificada sea verdadera.

En el bucle DO/WHILEse ejecutará el bloque de código una vez, antes de comprobar si la condición es verdadera, luego se repetira el bucle mientras la condición sea verdadera.

Break y Continue

Codigo Ej.

              	for (i = 0; i < 10; i++) {
            		    if (i === 3) { break; }  //se sale del bucle
            		    text += "El numero es " + i + "
";
            		}
            		
            		for (i = 0; i < 10; i++) {
            		    if (i === 3) { continue; } // se ignora el falor 3 y sigue la siguiente itera.
            		    text += "The number is " + i + "
";
            		}
Resumen

Break y continue

La sentencia 'break' rompe el bucle y continúa ejecutando el código después del bucle (si hay).

La sentencia 'continue' se rompe una iteración (en el bucle), si se produce una condición especificada, y continúa con la siguiente iteración del bucle.

Seccion #5 (Funciones)

Funciones

Codigo Ej.

              	function myFunction(a, b) { // a,b son parametros
		               return a * b;   // returna la multiplicacion
		            }
Resumen

Forma de la funcion

funciones declaradas no se ejecutan inmediatamente. Ellos son "salvados para su uso posterior", y se ejecutarán más adelante, cuando se invoca .

una funcion tiene la siguiente forma:

function functionName(parameters) {

codigo a ejecutar

}

Llamada de una funcion

Codigo Ej.

              	var person = {
            		  fullName: function() {
            			return this.firstName + " " + this.lastName;
          		    }
            		}
            		var person1 = {
            		    firstName:"John",
            		    lastName: "Doe",
            		}
            		var person2 = {
            		    firstName:"Mary",
            		    lastName: "Doe",
            		}
            		person.fullName.call(person1);  // retornara "John Doe"
Resumen

Llamado

En JavaScript todas las funciones son métodos de objeto.

En una definición de función, (this) se refiere al "dueño" de la función.

Con call () , un objeto puede utilizar un método que pertenece a otro objeto.
